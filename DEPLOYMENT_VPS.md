# 🚀 VPS Deployment Guide: Nginx, SSL & PM2

This guide explains how to deploy **Shaikh & Sons** to a VPS (Ubuntu) using Nginx as a reverse proxy, PM2 for process management, and Certbot for SSL.

---

## 1. Server Preparation

Update your server and install essential build tools:
```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y build-essential curl git
```

### Install Node.js (via NVM)
```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc
nvm install 20
nvm use 20
```

### Install PM2
```bash
npm install -g pm2
```

---

## 2. Database Setup (PostgreSQL)

Install PostgreSQL:
```bash
sudo apt install postgresql postgresql-contrib -y
```

Access Postgres and create a database/user:
```bash
sudo -u postgres psql
```
Inside the prompt:
```sql
CREATE DATABASE shaikh_db;
CREATE USER shaikh_user WITH PASSWORD 'your_strong_password';
GRANT ALL PRIVILEGES ON DATABASE shaikh_db TO shaikh_user;
\q
```

---

## 3. Project Setup

Clone your repository and install dependencies:
```bash
git clone <your-repo-url> shaikh-and-sons
cd shaikh-and-sons
npm install
```

### Environment Variables
Create a `.env.local` file:
```bash
nano .env.local
```
Paste your production values:
```env
DATABASE_URL=postgresql://shaikh_user:your_strong_password@localhost:5432/shaikh_db
IRON_SESSION_PASSWORD=at_least_32_characters_long_secret
OTP_JWT_SECRET=your_otp_signing_secret
ADMIN_EMAIL=admin@shaikhandsons.com
ADMIN_PASSWORD=your_secure_admin_password
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-password
```

---

## 4. Build & Database Migration

Run migrations to set up the schema:
```bash
npm run db:migrate
```

Build the Next.js application:
```bash
npm run build
```

---

## 5. PM2 Management

Start the application with PM2:
```bash
pm2 start npm --name "shaikh-app" -- start
```

Configure PM2 to start on boot:
```bash
pm2 startup
# Run the command generated by the output above
pm2 save
```

---

## 6. Nginx & SSL Configuration

Install Nginx:
```bash
sudo apt install nginx -y
```

### Create Nginx Config
```bash
sudo nano /etc/nginx/sites-available/shaikh.conf
```
Paste the following (Replace `subdomain.yourdomain.com` with your actual subdomain):
```nginx
server {
    listen 80;
    server_name subdomain.yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # IMPORTANT: Ensure public uploads are served correctly
    location /uploads/ {
        alias /var/www/shaikh-and-sons/public/uploads/;
        expires 30d;
        add_header Cache-Control "public, no-transform";
    }
}
```

Enable the site and restart Nginx:
```bash
sudo ln -s /etc/nginx/sites-available/shaikh.conf /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### SSL with Certbot
```bash
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d subdomain.yourdomain.com
```

## 📁 Storage Persistence Note

Since images are saved to `public/uploads/` on the VPS:
1.  **Permissions:** Ensure the user running the app has write access:
    ```bash
    chmod -R 755 public/uploads
    ```
2.  **Backups:** Regularly backup the `public/uploads` folder.
3.  **CI/CD:** If you use a CI/CD pipeline that deletes the folder on every deploy, you must move the `uploads` folder outside the project and use an Nginx `alias` or a symlink to keep files persistent.
